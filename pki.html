Project Overview

This project implements a working Public Key Infrastructure that simulates how HTTPS websites verify identity and establish trust using digital certificates. The system acts as a Certificate Authority that issues, validates, and revokes certificates using public-key cryptography.

Key Features

  • RSA-2048 key generation
  • Digital certificate issuance
  • Certificate verification
  • Digital signature validation
  • Certificate Revocation List (CRL)
  • Web-based verification interface

Security Concepts Demonstrated

  • Authentication
  • Integrity
  • Non-repudiation
  • Trust management

How It Works

1. Certificate Authority generates keys
2. User receives signed certificate
3. Verifier checks authenticity
4. If compromised, certificate is revoked
5. Verification fails after revocation
